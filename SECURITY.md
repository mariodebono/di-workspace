# Security Policy

## Supported Versions

Security fixes are provided for the latest published major version of each package.

| Package | Supported |
| --- | --- |
| `@mariodebono/di` | Latest major |
| `@mariodebono/di-config` | Latest major |
| `@mariodebono/di-app-migrations` | Latest major |
| `@mariodebono/di-electron` | Latest major |
| `@mariodebono/di-electron-i18n` | Latest major |
| `@mariodebono/di-drizzle-sqlite` | Latest major |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use GitHub private vulnerability reporting if enabled for this repository. If it is not enabled, contact the maintainer privately before disclosing details publicly.

Include:

- Affected package and version.
- A clear description of the issue.
- Reproduction steps or proof of concept, if available.
- Impact and affected environments.
- Any known mitigations.

## Disclosure Process

The maintainer will acknowledge valid reports, investigate impact, prepare a fix where appropriate, and coordinate publication of patched versions.

Public disclosure should wait until a fix or mitigation is available, unless there is active exploitation or another clear safety reason to disclose earlier.
